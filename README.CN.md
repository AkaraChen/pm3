## 为什么是 pm3？

PM2 很强大——但它的命令行界面像是为机器人设计的。每次操作都要记住晦涩的参数、拼接命令、解读神秘的输出。对于只想可靠运行应用的开发者来说，学习曲线过于陡峭。

**pm3 的核心理念很简单：启动一个进程不应该需要查阅文档。**

## 产品哲学

### 引导，而非猜测
不用再输入 `pm2 start app.js --name myapp --instances max --max-memory-restart 512M`，只需运行 `pm3 start`。回答几个简单的问题，搞定。我们处理参数，你专注于代码。

### 可发现性优于记忆
记不住扩展应用的具体命令？我们也记不住。这就是为什么 pm3 以交互方式展示你的选项——可用脚本、运行中的进程、配置选项——触手可及，无需查阅手册。

### 强大而不复杂
pm3 没有牺牲 PM2 的能力。集群模式、内存限制、监听模式、日志流、监控——都在。我们只是把复杂性隐藏在尊重你时间的对话式界面后面。

## 你能做什么

**启动与配置**
- 以交互方式启动 JavaScript 文件或 npm 脚本
- 通过提示配置集群模式、内存限制和监听模式
- 生成带有合理默认值的 ecosystem.config.js

**管理与控制**
- 通过可视化选择停止、重启或重载进程
- 干净地删除进程并带确认
- 在重启后保存和恢复进程列表

**监控与观察**
- 在整洁的格式化表格中查看所有进程
- 流式查看特定进程的日志
- 启动基于终端的监控仪表板

## 为谁而造

pm3 适合这样的开发者：
- 想要 PM2 的可靠性，但不想面对其学习曲线
- 喜欢交互式提示，而非记忆 CLI 参数
- 管理多个 Node.js 应用，需要快速操作
- 相信开发者工具应该平易近人，而非令人生畏

## 愿景

我们设想一个进程管理隐形的世界——部署和监控应用的感觉就像编写代码一样自然。pm3 是迈向那个未来的一步：以人为中心的设计包裹强大的基础设施。
